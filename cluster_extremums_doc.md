# Кластеры экстремумов (фиксированные значения) v12

Индикатор для TradingView на Pine Script v6, реализующий кластеризацию пивот-экстремумов с логикой запилов, offset-линиями и экспортом данных.

## Обзор

Индикатор ищет пивот-экстремумы на графике, группирует их в кластеры по ценовому порогу, отображает уровни поддержки/сопротивления и отслеживает «запилы» — касания уровней с последующим подтверждением.

**Ключевые возможности:**

- Поиск пивотных хаёв и лоёв
- Автоматическая кластеризация близких по цене экстремумов
- Отрисовка основных линий кластеров и offset-линий
- Заливка зоны между offset-линиями
- Логика запилов с подтверждением на следующих барах
- Алерты при касании offset-линий и при запилах
- JSON-экспорт всех данных в `log.info`
- Отображение абсолютных максимума и минимума за период

## Входные параметры

### Группа: Поиск экстремумов

| Параметр | Тип | По умолчанию | Описание |
|---|---|---|---|
| `leftBars` | int | 10 | Количество баров слева для определения пивота |
| `rightBars` | int | 10 | Количество баров справа для определения пивота |
| `lookback_bars` | int | 1000 | Глубина поиска абсолютных экстремумов (макс. 5000) |

### Группа: Кластеризация

| Параметр | Тип | По умолчанию | Описание |
|---|---|---|---|
| `cluster_threshold` | float | 0.01% | Максимальное отклонение цены внутри кластера (в %) |
| `min_cluster_size` | int | 3 | Минимальное число экстремумов для образования кластера |
| `use_only_recent` | bool | true | Ограничить выборку последними экстремумами |
| `recent_extremums_count` | int | 90 | Сколько последних экстремумов учитывать |
| `recalc_bars` | int | 500 | Пересчёт каждые N баров |
| `max_active_clusters` | int | 15 | Максимум одновременно активных кластеров |

### Группа: Offset линии

| Параметр | Тип | По умолчанию | Описание |
|---|---|---|---|
| `offset_percent` | float | 0.0005 | Отступ offset-линий от центра кластера (в долях) |

### Группа: Логика запилов

| Параметр | Тип | По умолчанию | Описание |
|---|---|---|---|
| `bars_offset` | int | 1 | Баров ожидания для подтверждения запила |
| `enable_breakthrough_logic` | bool | true | Включить/выключить всю логику запилов |
| `show_breakthrough_labels` | bool | true | Показывать метки при запилах |
| `enable_offset_alerts` | bool | true | Алерты при касании offset-линий |
| `max_breakthroughs` | int | 2 | Число запилов до деактивации кластера |
| `preserve_labels` | bool | false | Сохранять метки при пересчёте кластеров |

### Группа: Оповещения

| Параметр | Тип | По умолчанию | Описание |
|---|---|---|---|
| `alert_tolerance_percent` | float | 0.2% | Допуск для срабатывания алерта касания |

### Группа: Отображение

| Параметр | Тип | По умолчанию | Описание |
|---|---|---|---|

## Архитектура данных

### Тип `ExtremumData`

Хранит один пивот-экстремум.

| Поле | Тип | Описание |
|---|---|---|
| `price` | float | Цена пивота |
| `bar_index` | int | Индекс бара пивота |
| `bar_time` | int | Unix-время бара пивота |
| `is_clustered` | bool | Признак вхождения в кластер |

### Тип `UnifiedClusterData`

Хранит полное состояние одного кластера — данные, графику и статус запилов.

**Идентификация и геометрия:**

| Поле | Тип | Описание |
|---|---|---|
| `cluster_id` | string | Уникальный ID (цена + тип + время) |
| `avg_level` | float | Средняя цена кластера |
| `upper_offset_level` | float | Верхняя offset-линия |
| `lower_offset_level` | float | Нижняя offset-линия |
| `cluster_size` | int | Количество экстремумов в кластере |
| `is_high` | bool | true = сопротивление, false = поддержка |
| `start_bar` / `end_bar` | int | Диапазон баров кластера |
| `start_time` / `end_time` | int | Временной диапазон кластера |

**Состояние запилов:**

| Поле | Тип | Описание |
|---|---|---|
| `status` | string | `"active"` или `"deactivated"` |
| `breakthrough_count` | int | Текущее число запилов |
| `pending_confirmation_bar` | int | Бар, на котором ожидается подтверждение |
| `has_touched_level` | bool | Был ли уровень затронут |
| `came_from_correct_side` | bool | Подошла ли цена с нужной стороны |
| `last_breakthrough_bar` | int | Бар последнего подтверждённого запила |
| `breakthrough_in_progress` | bool | Флаг активного ожидания подтверждения |
| `upper_offset_touched` | bool | Касание верхней offset-линии |
| `lower_offset_touched` | bool | Касание нижней offset-линии |

**Графические объекты:**

| Поле | Тип | Описание |
|---|---|---|
| `cluster_line` | line | Основная линия уровня |
| `upper_offset_line` | line | Верхняя offset-линия |
| `lower_offset_line` | line | Нижняя offset-линия |
| `offset_fill` | linefill | Заливка между offset-линиями |
| `cluster_label` | label | Подпись размера кластера |
| `breakthrough_labels` | array\<label\> | Метки запилов |

### Тип `BreakthroughLabelInfo`

Дополнительная структура для управления метками запилов.

| Поле | Тип | Описание |
|---|---|---|
| `label_obj` | label | Ссылка на объект метки |
| `breakthrough_num` | int | Номер запила |
| `level` | float | Уровень кластера |
| `created_bar` | int | Бар создания метки |

## Основные функции

### Поиск и кластеризация

#### `find_absolute_extremums()`
Возвращает `[abs_high, abs_low]` — максимум и минимум по последним `lookback_bars` барам. Ограничение поиска: не более 4999 баров назад.

#### `find_clusters(extremums, is_high)`
Основная функция кластеризации. Алгоритм:
1. Берёт выборку `recent_extremums_count` последних экстремумов.
2. Итеративно расширяет кластер: для каждого экстремума ищет соседей в пределах `cluster_threshold` от текущего среднего.
3. Итерации повторяются до стабилизации (максимум 10 итераций).
4. Кластер принимается, если набрал `>= min_cluster_size` экстремумов.
5. Перед добавлением проверяет, нет ли похожего активного кластера (порог 80% от `cluster_threshold`).

#### `limit_array_size(extremums)`
Обрезает массив экстремумов до `recent_extremums_count` или 30 (при отключённом `use_only_recent`), удаляя самые старые.

### Управление кластерами

#### `create_cluster_graphics(cluster)`
Создаёт графику для кластера: основную линию, две offset-линии, заливку и метку. Все линии имеют `extend=extend.right` — продолжаются вправо до края графика.

Цвета: хай-кластеры — `#ffa500` (оранжевый), лоу-кластеры — `#0000ff` (синий).

#### `deactivate_cluster_immediately(cluster)`
Немедленно удаляет всю графику кластера (линии, заливку, метки, метки запилов), сбрасывает все флаги состояния и устанавливает `status = "deactivated"`.

#### `delete_cluster_graphics(cluster)`
Удаляет только графику (линии, заливку, метку), не трогая статус и флаги.

#### `cleanup_inactive_clusters()`
Удаляет из массива `unified_clusters` все элементы со статусом `"deactivated"`, проходя с конца.

#### `limit_graphics_objects()`
Вызывает `sync_cluster_array()`, затем при превышении `max_active_clusters` сортирует активные кластеры по `start_bar` (пузырьком) и деактивирует самые старые.

### Синхронизация

#### `is_cluster_truly_active(cluster)`
Проверяет реальную активность: статус `"active"` + существование основной линии + существование offset-линий (если включены).

#### `sync_cluster_array()`
Сверяет статус кластеров с наличием их графики. Кластеры, помеченные активными, но без графики, принудительно деактивируются. Затем удаляет деактивированные.

### Логика запилов

#### `check_level_touch(cluster_level)`
Возвращает `true`, если текущий бар пересекает уровень (с допуском 0.5%): `low <= level + tolerance` и `high >= level - tolerance`.

#### `check_approach_direction(cluster_level, is_high_cluster)`
Проверяет, что цена подошла к уровню с правильной стороны (снизу для сопротивления, сверху для поддержки) с допуском 0.5%.

#### `check_confirmation_simple(cluster)`
Проверяет подтверждение запила на баре `pending_confirmation_bar`: цена закрытия остаётся выше уровня (для хая) или ниже (для лоу) с допуском 0.2%.

#### `create_breakthrough_label(cluster, breakthrough_num)`
Создаёт цветную метку запила и отправляет `alert()`. Цвет меток хаёв: 1-й запил — лаймовый, последний — красный. Лоёв: 1-й — оранжевый, последний — тёмно-красный.

### Проверка offset-касаний

#### `check_offset_touch(cluster)`
Возвращает `[should_upper_alert, should_lower_alert]`.

Логика:
- Сброс флага касания, если цена отошла дальше 0.1% от offset-линии.
- Стандартное касание: бар пересекает offset-уровень.
- Гэп-касание: закрытие предыдущего бара по одну сторону, открытие текущего — по другую.
- Флаг устанавливается только если он ещё не был установлен (`upper/lower_offset_touched = false`).

#### `is_valid_offset_alert(cluster_level, current_price, tolerance_percent)`
Двойная проверка: цена в пределах `tolerance_percent` от уровня И не дальше 0.02% от него.

## Логика работы

### Поток исполнения на каждом баре

```
1. Определение пивотов (ta.pivothigh / ta.pivotlow)
   → Добавление в массивы high_extremums / low_extremums
   → Метки "H" (зелёные) и "L" (жёлтые) на графике

2. Обновление линий абсолютных экстремумов (только на barstate.islast)

3. Пересчёт кластеров (по расписанию recalc_bars или при движении >2%)
   → Очистка старой графики
   → find_clusters() для хаёв и лоёв
   → Восстановление состояния запилов (если preserve_labels=true)

4. Проверка offset-касаний (если enable_offset_alerts=true)
   → sync_cluster_array()
   → Для каждого активного кластера: check_offset_touch()
   → Генерация alert() при новом касании

5. Логика запилов (если enable_breakthrough_logic=true)
   → Проверка отложенных подтверждений
   → Поиск новых касаний уровней
   → Деактивация кластеров при достижении max_breakthroughs

6. Очистка старых меток (>300 баров)
7. Ограничение активных кластеров (limit_graphics_objects)
8. Сброс флагов алертов
```

### Пересчёт кластеров

Кластеры пересчитываются при выполнении хотя бы одного условия:
- `bar_index % recalc_bars == 0` и бар подтверждён
- Последний бар графика (`barstate.islast`)
- Изменение цены за 10 баров превысило 2%

### Деактивация кластера

Кластер деактивируется в двух случаях:
1. Превышено `max_active_clusters` — деактивируются самые старые.
2. Достигнут `max_breakthroughs` — все запилы отработаны.

## Алерты и оповещения

Индикатор генерирует `alert()` в четырёх ситуациях:

| Событие | Сообщение |
|---|---|
| Касание offset-линии | `"Касание верхней/нижней offset линии хай/лоу кластера! Уровень: ..."` |
| Первичное касание уровня | `"Касание обнаружено! HIGH/LOW кластер Уровень: ... Подтверждение на баре: ..."` |
| Подтверждённый запил | `"N-й запил хай/лоу уровня! Цена: ..., Уровень: ..."` |

> **Важно:** `alert_tolerance_percent` (по умолч. 0.2%) применяется как дополнительный фильтр для offset-касаний. Срабатывание происходит только при совпадении обоих условий: ценового порога И максимального расстояния 0.02%.

## Ограничения и константы

### Жёстко заданные стили

| Константа | Значение | Применение |
|---|---|---|
| `CLUSTER_LINE_WIDTH` | 2 | Ширина основной линии кластера |
| `OFFSET_LINE_WIDTH` | 1 | Ширина offset-линий |
| `ABS_LINE_WIDTH` | 3 | Ширина линий абсолютных экстремумов |
| `PIVOT_LABEL_SIZE` | `size.tiny` | Метки H/L на пивотах |
| `CLUSTER_LABEL_SIZE` | `size.small` | Подписи размера кластеров |
| `BREAKTHROUGH_LABEL_SIZE` | `size.normal` | Метки запилов |

### Жёсткие допуски внутри кода

| Использование | Значение |
|---|---|
| Проверка направления подхода | 0.5% от уровня |
| Проверка касания уровня | 0.5% от уровня |
| Подтверждение запила | 0.2% от уровня |
| Сброс флага offset-касания | 0.1% от уровня |
| Максимальное расстояние в `is_valid_offset_alert` | 0.02% от уровня |
| Порог похожести кластеров при добавлении | 80% от `cluster_threshold` |
| Триггер внепланового пересчёта | >2% за 10 баров |

### Платформенные ограничения

- `max_lines_count = 50` — лимит линий в одном индикаторе TradingView
- `max_labels_count = 500` — лимит меток
- Максимальная глубина поиска при `get_time_by_bar_index` — 5000 баров
- Метки запилов удаляются через 300 баров после создания
- Максимум итераций при расширении кластера — 10
